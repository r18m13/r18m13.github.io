UML-Based Robotic Trajectory Mapping and Video Analysis Tool

A UML-driven monocular visual odometry framework for frame-by-frame robot localization and 2D trajectory reconstruction from video data.

Overview

This project was developed to analyze robotic motion using monocular video data and structured software design. Traditional trajectory estimation systems often rely on depth sensors, stereo vision, or tightly coupled hardware setups, which increase system complexity. This work focuses on extracting reliable motion trajectories using only a single camera while maintaining a clean, extensible software architecture.

The system processes video sequences frame by frame to estimate relative robot motion and reconstruct its planar trajectory over time. UML-based modeling was used extensively to define system structure, data flow, and object responsibilities. The result is a scalable analysis tool that combines principled software engineering with monocular visual odometry for accurate 2D trajectory mapping.

Key Features

UML-Driven Architecture

System design is guided by UML class, sequence, and component diagrams to ensure modularity and maintainability.

Monocular Visual Odometry

Estimates relative robot motion from a single camera without relying on depth sensors or stereo input.

Frame-by-Frame Localization

Computes robot pose incrementally for each video frame, enabling continuous motion tracking.

2D Trajectory Reconstruction

Accumulates pose estimates to generate accurate planar trajectory maps from visual data.

Visual Motion Analysis

Feature detection and tracking provide insight into motion consistency and path behavior.

Technical Implementation

Architecture

The system follows a UML-based, object-oriented architecture built around a modular video processing pipeline. UML diagrams were used to define clear boundaries between components such as video input handling, feature extraction, motion estimation, and trajectory visualization.

Video sequences are processed sequentially, with each frame analyzed to detect and track visual features. Relative motion between frames is estimated using monocular visual odometry principles. These incremental pose estimates are accumulated in a global reference frame to reconstruct the robot’s 2D trajectory.

Example Python Code for Frame-by-Frame Motion Estimation:

import cv2
import numpy as np

cap = cv2.VideoCapture("robot_video.mp4")
ret, prev_frame = cap.read()
prev_gray = cv2.cvtColor(prev_frame, cv2.COLOR_BGR2GRAY)

while cap.isOpened():
  ret, frame = cap.read()
  if not ret: break
  gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
  # Feature tracking and relative motion estimation
  prev_gray = gray

The system is decomposed into the following core modules:

  • Video Input Module: Handles video loading, frame extraction, and preprocessing.
  • Feature Tracking Module: Detects and tracks visual features across frames.
  • Motion Estimation Module: Computes relative pose changes using monocular visual odometry.
  • Trajectory Mapping Module: Accumulates poses to form a 2D trajectory representation.
  • Visualization Module: Plots trajectories and overlays motion paths for analysis.

This modular architecture enables easy substitution of feature detectors, motion models, and visualization techniques.

Performance Optimizations

Several optimizations were applied to improve robustness and efficiency:

  • Efficient Feature Tracking: Reduces computational cost while maintaining stable localization.
  • Incremental Pose Integration: Updates trajectory estimates smoothly across frames.
  • Modular Pipeline Execution: Enables profiling and optimization of individual processing stages.

These optimizations ensure consistent trajectory estimation from long monocular video sequences.

Results & Impact

The system successfully reconstructs accurate 2D robot trajectories from monocular video input. The resulting trajectory plots clearly represent robot motion patterns and path progression, demonstrating reliable frame-to-frame localization and visual odometry.

By intentionally avoiding full 3D SLAM reconstruction, the project highlights the effectiveness of monocular visual odometry for planar motion analysis under minimal sensing constraints. The UML-driven design ensures the framework is easy to extend toward stereo vision, depth sensing, or full SLAM pipelines in future work.

Overall, the project demonstrates a strong integration of software architecture principles, computer vision, and robotic motion analysis, resulting in a clean, extensible, and technically sound trajectory mapping tool.
